# 项目汇报报告

## 项目名称

基于路侧多模态感知的低洼道路积水检测与短临预警系统。

当前版本：v0.9-streamlit-dashboard。

## 应用背景

雨季低洼路段容易形成积水，车辆经过时可能出现涉水熄火、道路拥堵、行人通行风险和财产损失。传统人工巡查存在发现不及时、覆盖范围有限、数据不连续等问题。

本项目面向路侧、路灯杆或监控杆部署场景，利用 LiDAR（激光雷达）和摄像头对低洼道路进行离线感知验证，形成“感知-建模-反演-预测-预警-审计”的完整工程原型。

## 解决的问题

本项目重点解决三个问题：

1. 如何把 LiDAR 点云、摄像头图像和 DEM（数字高程模型）组织成可复现的积水检测流程。
2. 如何从积水区域推算水深、面积、体积，并将结果输入短临预警链路。
3. 如何识别不可信结果，避免错误水深直接进入预警和行动建议。

## 系统总体方案

系统采用离线工程 MVP（最小可运行原型）方式实现。已有 rosbag（ROS 数据包）作为输入，不启动实时 LiDAR、摄像头或 ROS 节点。核心流程如下：

S1 多模态采集 -> S2 ground DEM（地面数字高程模型） -> S3 mask（掩膜） -> S4 水深反演 -> S5 面积体积 -> S6 气象修正 -> S7 三层混合推理 -> S8 分级预警与审计日志 -> Agent（智能体）调度 -> dashboard（可视化看板）。

## S1-S8 技术路线

### S1 多模态采集

采集 LiDAR 点云和摄像头图像，形成离线 rosbag（ROS 数据包）。当前项目不依赖实时设备运行，便于复现实验和课堂演示。

### S2 ground DEM 构建

利用无水场景点云构建 ground DEM（地面数字高程模型），作为后续水深反演的空间基准。项目区分了初版场景高度图和真正用于水深计算的地面高程基准。

### S3 积水 mask

实现了两类 mask（掩膜）：图像空间人工 polygon mask，以及 playground_pit 场景的 DEM-space mask（DEM 栅格掩膜）。后者用于 S4-real-B 边界水位线反演。

### S4 水深反演

项目包含三条路径：

- S4-MVP configured_depth：配置水深，用于验证完整工程链路，不代表真实最终水深测量。
- S4-real-A surface DEM：用 LiDAR 当前表面 DEM 与 ground DEM 做差，直接反演水深。
- S4-real-B boundary waterline：当 LiDAR 难以稳定扫到浅水水面时，尝试用积水边界和 ground DEM 估计水位线。

### S5 面积体积计算

基于水深图和有效 mask 计算积水面积、体积、最大水深、平均水深等指标。

### S6 气象修正

当前使用 offline mock weather（离线模拟气象）验证气象修正因子进入推理链路。后续可以替换为真实气象 API。

### S7 三层混合推理

S7-A 是确定性规则引擎，S7-B 是历史案例检索修正，S7-C 是简化物理约束校验。三层组合后输出最终短临预测结果。

### S8 分级预警与审计日志

系统输出分级预警、行动建议、Markdown 报告和审计日志，并通过 SQLite 数据库保存 Agent（智能体）运行摘要。

## 当前已完成内容

项目已完成 S1-S8 离线 MVP、Agent 调度、SQLite 审计数据库、Streamlit dashboard（可视化看板）、GitHub 公开仓库同步，以及 S4-real 的质量诊断与 quality gate（质量门控）。

## 关键实验结果

1. 宿舍纸箱/可控场景可以完整展示从感知到预警的工程管线。
2. 宿舍 39cm 可控场景中，S4-real-A surface DEM 直接反演结果较接近人工水深，说明链路具备可行性。
3. 宿舍 13cm 场景出现明显高估，说明浅水和点云覆盖不足会影响反演精度。
4. 操场坑洼 6cm 场景中，S4-real-A 和 S4-real-B 均被 quality gate（质量门控）判定为 reject（拒绝进入后续链路）。
5. Streamlit dashboard（可视化看板）已完成，可用于老师汇报、同学协作和项目演示。

## 当前问题分析

当前主要问题不是工程链路没有跑通，而是浅水场景的数据质量和标定条件不足。LiDAR 对浅水水面回波不稳定，操场坑洼边界不清晰，点云覆盖稀疏，mask（掩膜）边界高程离散较大，导致水位估计不可靠。

## 为什么操场坑洼 6cm 不准

操场坑洼 6cm 场景不准的主要原因包括：

- 浅水厚度小，LiDAR 水面回波弱且不稳定。
- 坑洼边界不是规则容器边界，ground DEM 上边界高程离散较大。
- 当前 DEM-space mask 虽然 shape 正确，但诊断显示边界 height std（高程标准差）较大，point_count（点云数量）中位数为 0。
- 场地简陋、边界不清、采集角度和点云覆盖不足，使得边界水位线估计不稳定。

## 为什么这不是项目失败

这不是项目失败，而是工程系统应有的质量识别能力发挥了作用。真实部署系统不能只追求输出一个水深数值，更重要的是判断这个数值是否可信。本项目没有伪造 6cm 结果，也没有为了贴近已知水深强行调参，而是用 quality gate（质量门控）把不可信结果 reject（拒绝进入后续链路）。

## 质量门控的工程意义

quality gate（质量门控）的意义在于：

- 防止错误水深进入 S5 面积体积计算和 S8 预警链路。
- 将“算法结果”与“结果可信度”分开管理。
- 对低覆盖率、高误差、极端异常值和边界高程不稳定进行显式判断。
- 为后续现场部署提供安全边界，避免系统在数据质量不足时给出误导性预警。

## 项目创新点

1. 完成从 LiDAR、摄像头、DEM 到预警报告的端到端离线工程原型。
2. 同时实现 S4-MVP、S4-real-A 和 S4-real-B 三条水深反演路径。
3. 引入质量诊断和 quality gate（质量门控），能够拒绝不可信结果。
4. 实现 S7-A/B/C 三层混合推理和 S8 分级预警。
5. 引入 Agent（智能体）调度和 SQLite 审计数据库，具备可追溯性。
6. 提供 Streamlit dashboard（可视化看板），便于老师汇报和项目演示。

## 后续完善方向

- 重新采集更规范的低洼场景数据，包括稳定 dry baseline（无水基准）和不同深度水样本。
- 做 LiDAR 与摄像头外参标定，提高 mask（掩膜）到 DEM（数字高程模型）的映射精度。
- 优化浅水场景下的点云滤波、水面回波识别和边界水位线估计。
- 引入真实气象 API 和连续时序数据，替代当前 offline mock weather（离线模拟气象）。
- 将离线 pipeline 逐步扩展为 rosbag replay 和实时部署版本。

## 当前阶段结论

本项目已经完成课程/比赛展示所需的主要工程原型。系统可以展示完整管线、关键算法模块、质量诊断、质量门控、预警报告、Agent 调度和 dashboard（可视化看板）。当前浅水真实精度仍需进一步采集与标定优化，但系统已经具备识别不可信结果并阻断错误预警的工程能力。
