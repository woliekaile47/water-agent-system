# 演示操作说明

## 进入项目目录

```bash
cd ~/water_agent_ws/water_agent_system
```

## 运行健康检查

如果项目中已存在健康检查脚本，可运行：

```bash
python3 scripts/check_project_health.py
```

健康检查主要用于确认配置文件、输出目录、关键 JSON 和图像是否存在。若某些实验输出缺失，应先按对应阶段重新运行离线 pipeline。

## 运行离线 demo

如果需要演示完整离线 MVP 链路，可运行：

```bash
bash scripts/run_full_offline_demo.sh
```

该 demo 不需要启动实时 LiDAR、摄像头或 ROS 节点。它读取已有离线结果或离线配置，展示 S4 到 S8 以及 Agent（智能体）调度流程。

## 启动 Streamlit dashboard（可视化看板）

```bash
streamlit run dashboard/app.py
```

如果浏览器没有自动打开，请访问 Streamlit 输出的地址，通常是：

```text
http://localhost:8501
```

如果在虚拟机中运行，需要根据网络设置访问对应虚拟机 IP 和端口。

## 演示时建议点击的页面

### 1. 首页：项目概览

说明项目应用场景：雨季低洼道路积水，路侧设备通过 LiDAR（激光雷达）和摄像头监测水深、面积、体积和未来风险。强调系统是从 S1 到 S8 的完整工程原型。

### 2. S4-MVP：configured_depth 管线

说明 configured_depth 是 MVP 管线验证，不是真实最终水深测量。它的作用是证明 mask（掩膜）到 DEM（数字高程模型）到水深图、面积体积、预警链路可以跑通。

### 3. S4-real-A：surface DEM 直接反演

展示宿舍 13cm 和 39cm 场景。重点解释 39cm 结果较好，说明 LiDAR surface DEM 直接反演具备可行性；13cm 高估，说明浅水和覆盖不足会影响精度。

### 4. S4-real：质量门控

展示 playground_pit_water_sim_6cm_001 的 quality gate（质量门控）。重点解释 quality_status = reject（拒绝进入后续链路），can_enter_s5_s8_warning_chain = False。说明不可信结果不会进入正式预警链路。

### 5. S4-real-B：边界水位线反演

说明这是 fallback（备用）路线：当 LiDAR 难以直接检测浅水水面高程时，尝试通过积水边界和 ground DEM（地面数字高程模型）估计水位线。当前操场 6cm 仍被 reject，原因是 mask 边界高程离散大。

### 6. DEM-space mask 诊断

展示 mask（掩膜）诊断图。重点说明当前 mask shape 正确，但 boundary height std（边界高程标准差）较大，point_count（点云数量）median 为 0，说明数据采集条件和边界质量不足，需要人工精修。

### 7. S7/S8：短临预警链路

展示 deterministic forecast（确定性预测）、physical constraint（物理约束）和 warning summary（预警摘要）。说明这部分是完整短临预警链路，但如果 S4-real 被质量门控 reject，则真实 S4-real 结果不能直接进入正式预警链路。

### 8. Agent 与审计

展示 Agent（智能体）运行摘要和 SQLite 审计数据库路径。说明系统具备调度、结果汇总和审计记录能力。

## 演示注意事项

- 不需要启动实时 LiDAR。
- 不需要启动实时摄像头。
- 不需要启动 ROS 节点。
- 不需要 rosbag replay。
- 不要现场修改 rosbag 或核心算法。
- 如果某个 dashboard 页面显示 warning（缺少文件），说明该输出尚未生成，不代表系统崩溃。
